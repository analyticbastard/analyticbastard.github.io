Databases

Mar 8th, 2015 9:13 pm

I have used Datomic extensively to build the repository for the core product at Erudine. I implemented features that required low-level Datomic capabilities, especially dealing with transactions, creating transaction attributes to mark type and id so that they could be reverted, independently of how many application entities each affected. Also, a very interesting feature that we achieved was transitivity between nodes using Datomic rules, so that we could be able to traverse nodes in the graph of application entities graph.

I have used MySQL and PostgresQL for years and am acquainted with inner, left, right and outher join operations, and prepared statements to query databases from code.
