Handling low level byte frames in Clojure with Gloss

Gloss is a (another) fine library by Zach Tellman which handles low level byte manipulation in Clojure. Sounds weird? It is!

How do we functionally handle system level byte frames that are otherwise naturally manipulated in assembler or C? Gloss comes to rescue.

Imagine that we are given a short integer (16 bit) (def intvalue 1) and need to invert the byte positions. How we proceed with Gloss is:

  • Define a codec to handle
    (require '[gloss.core :as core :refer [defcodec]])
    (defcodec mycodec
        [:uint16 :uint16])
  • Encode the value into a frame, which is (elegantly) a vector of values.
   (require '[gloss.io :as io])

   (assert (= intvalue 0x1))
   (def frame (io/encode mycodec intvalue))
  • Manually invert the bytes as treating any Clojure collection
   (def outframe (reverse frame))
  • Decode the frame with the codec again, to get the integer value
   (def outvalue (io/decode mycodec outframe))
   (assert (= outvalue 0x256))

This is very basic but you get the idea. A very useful tutorial, more than the (unexistent) official documentation, is Gloss by Example, where the author builds a basic RFC 5322 header-codec.
